|image1|

` <#what-is-canopsis->`__\ What is Canopsis ?
---------------------------------------------

`Canopsis <http://canopsis.org>`__ is an open-source
`hypervisor <http://www.capensis.fr/solutions/hypervision/>`__ whose
goal is to
`aggregate/consolidate </capensis/canopsis/wiki/consolidation>`__
information and events (containing
`metrics </capensis/canopsis/wiki/Metrics>`__ of different types such as
performance, availability, etc.) coming from multiple sources to create
a global solution for `monitoring </capensis/canopsis/wiki/Dashboard>`__
and `administrating </capensis/canopsis/wiki/engines>`__ resources.

Built to last on top of `proven Open Source technologies by and for all
IT professionals <http://www.capensis.fr/solutions/supervision/>`__. It
is an event based architecture and it is modular by design. Plug your
infrastructure tools like ``Syslog``, ``Nagios``,
```Shinken`` <https://github.com/naparuba/shinken>`__, ``...`` to
`Canopsis <http://canopsis.org>`__ and you're ready to go.

A `Glossary </capensis/canopsis/wiki/Glossary>`__ page is also given for
better descriptions about canopsis concepts.

` <#how-to-try->`__\ How to try ?
---------------------------------

You can try Canopsis on demo platform:

-  Master branch (stable):
   `http://demo.canopsis.org <http://demo.canopsis.org>`__
-  Devel branch (unstable):
   `http://demo-devel.canopsis.org <http://demo-devel.canopsis.org>`__

` <#how-to-install->`__\ How to install ?
-----------------------------------------

Two ways for install:

Easy way (on ``Debian 6 & ulterior``, ``CentOS 6``,
``Ubuntu 12.04 & ulterior`` 64Bits):

-  `https://github.com/capensis/canopsis/wiki/Install-from-packages <https://github.com/capensis/canopsis/wiki/Install-from-packages>`__

And developers way (or unsupported environment):

-  `https://github.com/capensis/canopsis/wiki/Install-from-sources <https://github.com/capensis/canopsis/wiki/Install-from-sources>`__

` <#how-to-connect->`__\ How to connect ?
-----------------------------------------

Now you are ready to deal with Canopsis.

You can connect your ``Nagios`` (or ``Icinga``):

-  `https://github.com/capensis/canopsis-nagios <https://github.com/capensis/canopsis-nagios>`__

Or your ```Shinken`` <https://github.com/naparuba/shinken>`__:

-  `https://github.com/capensis/canopsis/wiki/Shinken <https://github.com/capensis/canopsis/wiki/Shinken>`__

` <#how-to-use->`__\ How to use ?
---------------------------------

A `Dashboard </capensis/canopsis/wiki/Dashboard>`__ containing
`views </capensis/canopsis/wiki/Views>`__ and
`Widgets </capensis/canopsis/wiki/Widgets>`__ is provided in order to
monitor and to `administer </capensis/canopsis/wiki/engines>`__
resources.
